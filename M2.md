# Crooshubext — метрики для Grafana

**Сервис:** Crooshubext — маршрутизатор запросов к AI-агентам.

Принимает входящие вызовы по HTTP, определяет нужного агента и доставляет запрос либо по REST, либо через Kafka. Возвращает ответ агента вызывающей стороне.

**Формат метрик:** Prometheus. Неймспейс — `crooshubext`.

---

## Входящие запросы — `/api/v1/invoke`

Единственный публичный эндпоинт. Через него клиенты инициируют вызов агента.

---

### `crooshubext_invoke_requests_total`
**Тип:** Counter
**Описание:** Общее количество входящих запросов на `/api/v1/invoke`.

| Label | Значения | Описание |
|---|---|---|
| `status` | `success`, `error` | Завершился ли вызов успешно |
| `agent_id` | `agent-abc`, `agent-xyz`, ... | К какому агенту был направлен запрос |
| `routing_mode` | `rest`, `kafka` | Каким способом был доставлен запрос агенту |

**Панель:** Rate входящих вызовов, разбивка по агентам и способу доставки.
```
rate(crooshubext_invoke_requests_total[1m])
```

**Error rate (%):**
```
100 * sum(rate(crooshubext_invoke_requests_total{status="error"}[5m]))
    / sum(rate(crooshubext_invoke_requests_total[5m]))
```

---

### `crooshubext_invoke_duration_seconds`
**Тип:** Histogram
**Описание:** Полное время обработки запроса — от получения до отправки ответа клиенту. Включает время ожидания ответа агента.

| Label | Значения | Описание |
|---|---|---|
| `agent_id` | `agent-abc`, `agent-xyz`, ... | К какому агенту шёл запрос |
| `routing_mode` | `rest`, `kafka` | Способ доставки |
| `status` | `success`, `error` | Итог вызова |

**Buckets:** `0.1, 0.25, 0.5, 1, 2, 5, 10, 30, 60` секунд (AI-агенты могут отвечать долго).

**Панель:** Latency p50/p95/p99 по агентам:
```
histogram_quantile(0.99,
  sum by (agent_id, le) (
    rate(crooshubext_invoke_duration_seconds_bucket[5m])
  )
)
```

---

### `crooshubext_invoke_in_flight`
**Тип:** Gauge
**Описание:** Сколько запросов сейчас ожидают ответа агента (в обработке).

| Label | Значения | Описание |
|---|---|---|
| `agent_id` | `agent-abc`, `agent-xyz`, ... | К какому агенту ждут ответа |

**Панель:** Текущая нагрузка. Позволяет видеть, где скапливаются запросы.
```
crooshubext_invoke_in_flight
```

---

## Исходящие вызовы агентов

Сервис вызывает агентов двумя способами: синхронно по REST или асинхронно через Kafka. Метрики описываются отдельно для каждого способа.

---

### REST-вызовы агентов

#### `crooshubext_agent_rest_requests_total`
**Тип:** Counter
**Описание:** Количество REST-вызовов к агентам.

| Label | Значения | Описание |
|---|---|---|
| `agent_id` | `agent-abc`, `agent-xyz`, ... | Идентификатор агента |
| `status_code` | `200`, `400`, `500`, `503`, ... | HTTP-код ответа агента |

**Панель:** Error rate по каждому агенту:
```
100 * sum by (agent_id) (
    rate(crooshubext_agent_rest_requests_total{status_code=~"5.."}[5m])
  )
  / sum by (agent_id) (
    rate(crooshubext_agent_rest_requests_total[5m])
  )
```

---

#### `crooshubext_agent_rest_duration_seconds`
**Тип:** Histogram
**Описание:** Время ответа агента по REST (от отправки запроса до получения ответа).

| Label | Значения | Описание |
|---|---|---|
| `agent_id` | `agent-abc`, `agent-xyz`, ... | Идентификатор агента |
| `status_code` | `200`, `500`, ... | Код ответа |

**Buckets:** `0.1, 0.25, 0.5, 1, 2, 5, 10, 30, 60` секунд.

**Панель:** p99 latency каждого агента:
```
histogram_quantile(0.99,
  sum by (agent_id, le) (
    rate(crooshubext_agent_rest_duration_seconds_bucket[5m])
  )
)
```

---

### Kafka-вызовы агентов (Producer — `topic-in`)

Когда `routing_mode = kafka`, сервис публикует задачу в `topic-in` и ожидает ответа из `topic-out`.

#### `crooshubext_kafka_producer_messages_total`
**Тип:** Counter
**Описание:** Количество сообщений, отправленных в `topic-in`.

| Label | Значения | Описание |
|---|---|---|
| `topic` | `topic-in` | Топик назначения |
| `agent_id` | `agent-abc`, `agent-xyz`, ... | Для какого агента задача |
| `status` | `success`, `error` | Удалось ли отправить |

**Панель:** Throughput и error rate продюсера:
```
rate(crooshubext_kafka_producer_messages_total[1m])
```

```
100 * sum(rate(crooshubext_kafka_producer_messages_total{status="error"}[5m]))
    / sum(rate(crooshubext_kafka_producer_messages_total[5m]))
```

---

#### `crooshubext_kafka_producer_duration_seconds`
**Тип:** Histogram
**Описание:** Время от вызова produce до подтверждения от брокера (ack).

| Label | Значения | Описание |
|---|---|---|
| `topic` | `topic-in` | Топик |

**Buckets:** `0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1` секунд.

**Панель:** p99 задержки публикации — если растёт, проблема на стороне брокера:
```
histogram_quantile(0.99,
  rate(crooshubext_kafka_producer_duration_seconds_bucket[5m])
)
```

---

### Kafka-получение ответов (Consumer — `topic-out`)

#### `crooshubext_kafka_consumer_messages_total`
**Тип:** Counter
**Описание:** Количество сообщений (ответов агентов), полученных из `topic-out`.

| Label | Значения | Описание |
|---|---|---|
| `topic` | `topic-out` | Топик |
| `agent_id` | `agent-abc`, `agent-xyz`, ... | От какого агента ответ |
| `status` | `success`, `error` | Успешно ли обработан ответ |

**Панель:** Rate входящих ответов от агентов через Kafka:
```
rate(crooshubext_kafka_consumer_messages_total[1m])
```

---

#### `crooshubext_kafka_consumer_processing_duration_seconds`
**Тип:** Histogram
**Описание:** Время от получения сообщения из `topic-out` до коммита оффсета (время обработки одного ответа агента).

| Label | Значения | Описание |
|---|---|---|
| `topic` | `topic-out` | Топик |

**Buckets:** `0.001, 0.005, 0.01, 0.05, 0.1, 0.5, 1` секунд.

**Панель:**
```
histogram_quantile(0.99,
  rate(crooshubext_kafka_consumer_processing_duration_seconds_bucket[5m])
)
```

---

#### `crooshubext_kafka_consumer_lag`
**Тип:** Gauge
**Описание:** Отставание консюмера `topic-out` — разница между последним offset в партиции и текущим прочитанным. Рост лага означает, что агенты отвечают быстрее, чем сервис успевает забирать ответы.

| Label | Значения | Описание |
|---|---|---|
| `topic` | `topic-out` | Топик |
| `partition` | `0`, `1`, ... | Номер партиции |
| `consumer_group` | `crooshubext-cg` | Группа консюмера |

> Эту метрику рекомендуется собирать через `kafka-exporter` на уровне инфраструктуры — уточнить у команды DevOps.

**Панель:** Суммарный лаг по всем партициям:
```
sum by (topic) (crooshubext_kafka_consumer_lag)
```

**Алерт:** Если лаг стабильно растёт — сервис не справляется с нагрузкой.

## Рекомендуемые панели дашборда

| Панель | Метрики | Что показывает |
|---|---|---|
| **Invoke RPS** | `invoke_requests_total` | Входящий трафик, разбивка по агентам и routing_mode |
| **Invoke Error Rate** | `invoke_requests_total` | % ошибок входящих запросов |
| **Invoke Latency** | `invoke_duration_seconds` | p50/p95/p99 по каждому агенту |
| **In-flight Requests** | `invoke_in_flight` | Сколько запросов сейчас висят в ожидании |
| **Agent REST Health** | `agent_rest_requests_total`, `agent_rest_duration_seconds` | Ошибки и latency по REST-агентам |
| **Kafka Producer** | `kafka_producer_messages_total`, `kafka_producer_duration_seconds` | Throughput и задержка публикации в topic-in |
| **Kafka Consumer Lag** | `kafka_consumer_lag` | Лаг по topic-out — риск накопления очереди |
| **Kafka Consumer RPS** | `kafka_consumer_messages_total` | Скорость обработки ответов агентов |

---

## Алерты

| Алерт | Условие | Severity |
|---|---|---|
| Высокий error rate на /invoke | error rate > 5% за 5 мин | critical |
| Деградация latency | p99 invoke > 30s за 5 мин | warning |
| REST-агент недоступен | error rate агента > 10% за 5 мин | critical |
| Kafka producer не пишет | rate = 0 при наличии in-flight Kafka-запросов | critical |
| Kafka consumer lag растёт | lag > 500 и d/dt > 0 за 10 мин | warning |
| Накопился большой lag | lag > 2000 | critical |
